# ♿ Accessibility Auditor (WCAG 2.1 AA)

## Цел
Проверка за достъпност на сайта за хора с увреждания, следвайки WCAG 2.1 Level AA стандарти.

## Стъпки

### 1. Семантичен HTML (WCAG 1.3.1)
- Провери `<main>`, `<nav>`, `<header>`, `<footer>`, `<aside>` — използвани ли са?
- Провери heading hierarchy: има ли единствен `<h1>`? Правилна ли е йерархията h1→h2→h3?
- Провери `<button>` vs `<div onclick>` — интерактивни елементи трябва да са native бутони
- Провери `<table>` за `<th>`, `<caption>`, `scope` атрибути
- Провери lists: `<ul>/<ol>/<li>` за навигация и списъци

### 2. Клавиатурна навигация (WCAG 2.1.1)
- Отвори сайта в браузъра
- Натисни Tab многократно — може ли да се достигне всеки интерактивен елемент?
- Провери дали фокусът е видим (`:focus` стилове в styles.css)
- Провери модали: затваря ли се с Escape? Заключен ли е фокусът (focus trap)?
- Провери dropdown менюта — достъпни ли са с клавиатура?
- Провери skip-to-content link

### 3. Aria атрибути (WCAG 4.1.2)
- Търси `aria-label`, `aria-labelledby`, `aria-describedby` в index.html
- Провери SVG иконите — имат ли `aria-hidden="true"` или `<title>`?
- Провери динамични области: `aria-live` за toast нотификации и cart badge
- Провери модали: `role="dialog"`, `aria-modal="true"`
- Провери бутони без текст — имат ли `aria-label`?

### 4. Цветов контраст (WCAG 1.4.3)
- Прочети CSS variables в `:root` блока на styles.css
- Калкулирай контраст ratio за:
  - `--text` (#1a1a1a) върху `--bg` (#f8f9fa) → трябва ≥ 4.5:1
  - `--primary` (#bd1105) върху `--white` (#fff) → трябва ≥ 4.5:1
  - `--muted` (#9ca3af) върху `--bg` (#f8f9fa) → трябва ≥ 4.5:1
  - Бял текст върху `--primary` бутони → трябва ≥ 4.5:1
- Провери sale/promo badge цветове за достатъчен контраст
- Провери disabled state цветове

### 5. Изображения и медия (WCAG 1.1.1)
- Провери всички `<img>` — имат ли `alt` атрибут?
- Провери emoji fallback изображения — достъпни ли са?
- Провери product card images за смислени alt текстове
- Провери SVG sprite — всяка икона трябва да е или `aria-hidden` или да има label

### 6. Форми (WCAG 1.3.1, 3.3.2)
- Провери всички `<input>` — имат ли `<label>` или `aria-label`?
- Провери error messages — свързани ли са с input полетата?
- Провери autocomplete атрибути за лични данни (name, email, tel)
- Провери required полета — обозначени ли са визуално И програмно?

### 7. Responsive & Motion (WCAG 1.4.4, 2.3.1)
- Провери zoom до 200% — съдържанието чете ли се?
- Провери `prefers-reduced-motion` media query в CSS
- Провери за flashing content (>3 flashes/second)

## Severity
- 🔴 **A (Critical)** — Задължителни по закон в ЕС (European Accessibility Act)
- 🟠 **AA (Required)** — Стандартно ниво за уеб сайтове
- 🟡 **AAA (Ideal)** — Най-високо ниво, препоръчително
