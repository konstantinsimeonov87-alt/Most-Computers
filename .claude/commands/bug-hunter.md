# 🐛 Агент: Бъг Хънтър (Bug Hunter)

Този workflow систематично търси бъгове в сайта чрез код анализ и реално тестване в браузъра.
**Не спирай докато не намериш и оправиш всички критични бъгове.**

## Стъпки

### 1. Прочети кода — статичен анализ

**JavaScript** — прочети source файловете в `js/` (вж. `js/_load-order.txt`). `app.js` е генериран bundle — fix-ове трябва да отиват в `js/`, не в него. Търси:
- `null` / `undefined` reference грешки — липсващи null checks
- Off-by-one грешки в loops
- Race conditions в async операции
- Неправилно използване на `==` вместо `===`
- Missing `break` в switch statements
- localStorage/sessionStorage достъп без try/catch
- Event listeners, които не се cleanup-ват

**HTML** — прочети `index.html` и търси:
- Дублирани `id` атрибути
- Неправилно nested форм елементи
- Missing `type="button"` на бутони във форми

**CSS** — прочети `styles.css` и търси:
- `z-index` конфликти — особено между `position: fixed` елементи и footer
- `overflow: hidden` което скрива важно съдържание
- `position: fixed/sticky` проблеми на mobile Safari
- `100vh` проблем на мобилни браузъри (адрес барът)
- Footer layout проблеми — grid/flex разкъсване на desktop

### 2. Визуален одит — Desktop (1280px+)

**Отвори `index.html` в браузъра и провери визуално всеки раздел:**

#### Хедър и навигация
- [ ] Лого, търсачка, икони — правилно наредени на едно ниво
- [ ] Sticky header — не припокрива съдържанието при scroll
- [ ] Мега меню — отваря се под хедъра, не извън viewport-а

#### Главна страница
- [ ] Hero секция — пълна ширина, текстът е четим
- [ ] Продуктови карти — равномерна решетка, снимките не са разтегнати
- [ ] Sidebar (ако е видим) — не излиза извън container-а

#### Footer — **приоритет**
- [ ] Отвори `index.html`, scroll до дъното
- [ ] Footer се вижда ли изцяло? Или е отрязан/скрит?
- [ ] Колоните (лого, навигация, контакти) — правилно наредени side by side
- [ ] Текстът в footer не е припокрит от друг елемент
- [ ] Footer не е зад overlay или fixed елемент
- [ ] Footer background цвят е правилен
- [ ] Copyright ред е видим в дъното
- [ ] Прочети CSS за `.footer`, `footer`, `.site-footer` — намери z-index, position, overflow стойностите
- [ ] Ако footer-ът е счупен: намери причината в CSS/JS и я оправи незабавно

#### Категорийна страница (cat-page overlay)
- [ ] Кликни на категория — overlay се отваря над footer-а
- [ ] Sidebar с филтри — вляво, scroll-ва независимо
- [ ] Продуктова решетка — вдясно, правилна
- [ ] Затвори overlay — footer се вижда нормално

### 3. Визуален одит — Mobile (375px — iPhone SE)

**Смени браузъра на mobile viewport (DevTools → Toggle Device Toolbar → 375px):**

#### Хедър
- [ ] Hamburger меню е видимо
- [ ] Търсачката не излиза извън екрана
- [ ] Tap targets са минимум 44×44px

#### Продуктови карти
- [ ] 1 или 2 колони на мобилен — не е наредено хаотично
- [ ] Цените са четими (не отрязани)
- [ ] "Добави в кошница" бутонът е достъпен

#### Footer на мобилен
- [ ] Footer колоните са наредени вертикално (stack)
- [ ] Линковете са достатъчно широки за tap
- [ ] Не е скрит или отрязан

#### Мобилно меню
- [ ] Hamburger → меню се отваря от ляво/дясно
- [ ] Backdrop click → затваря менюто
- [ ] Body scroll lock — не може да scroll-ва зад менюто

### 4. Функционално тестване

#### Кошница
- [ ] Добави продукт → badge се обновява
- [ ] Добави същия продукт 2 пъти → количеството расте
- [ ] Премахни продукт → цената се преизчислява
- [ ] Количество 0 → продуктът се премахва

#### Търсене
- [ ] Въведи текст → live резултати
- [ ] Търси несъществуващ продукт → "няма резултати"
- [ ] Специални символи (`<script>`) → няма XSS

#### Филтри
- [ ] Филтър по категория → показва правилните продукти
- [ ] Сортиране → правилен ред
- [ ] Комбинация от филтри → работи

### 5. Console мониторинг
- Отвори DevTools → Console
- Провери за грешки при: зареждане, навигация, добавяне в кошница, отваряне на модали
- Логвай всички `Error` и `Warning`

### 6. Оправи намерените бъгове

**За всеки открит бъг:**
1. Идентифицирай причината в source файловете (`js/`, `styles.css`, `index.html`)
2. Приложи fix директно
3. За JS промени: след fix rebuild с `node build.js`
4. Пусни `npm test` — трябва да минат всички тестове
5. Потвърди fix-а визуално в браузъра

**Не спирай докато не са оправени всички 🔴 Critical бъгове.**

### 7. Създай Bug Report

Създай artifact `bug_report.md` с:
- 🔴 **Critical** — блокиращи бъгове
- 🟠 **Major** — функционални проблеми
- 🟡 **Minor** — козметични или edge case проблеми
- 🟢 **Enhancement** — не е бъг, но може да е по-добре

За всеки бъг:
- ID (BUG-001, BUG-002...)
- Стъпки за възпроизвеждане
- Очаквано поведение
- Действително поведение
- Приложен fix (файл, ред, промяна)
