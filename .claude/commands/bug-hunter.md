# 🐛 Агент: Бъг Хънтър (Bug Hunter)

Този workflow систематично търси бъгове в сайта чрез код анализ и реално тестване в браузъра.

## Стъпки

### 1. Статичен анализ за потенциални бъгове
- Прочети source файловете в `js/` (вж. `js/_load-order.txt`). `app.js` е генериран bundle — fix-ове трябва да отиват в `js/`, не в него. Търси в source файловете:
  - `null` / `undefined` reference грешки — липсващи null checks
  - Off-by-one грешки в loops
  - Race conditions в async операции
  - Неправилно използване на `==` вместо `===`
  - Missing `break` в switch statements
  - Prototype pollution рискове
  - Regex грешки (ReDoS уязвимости)
  - localStorage/sessionStorage достъп без try/catch
  - Math грешки (делене на нула, floating point)
  - Event listeners, които не се cleanup-ват

- Прочети `index.html` и търси:
  - Дублирани `id` атрибути
  - Неправилно nested форм елементи
  - Missing `type="button"` на бутони във форми
  - Broken links (href="#" without handler)

- Прочети `styles.css` и търси:
  - `z-index` конфликти
  - `overflow: hidden` което скрива важно съдържание
  - `position: fixed/sticky` проблеми на mobile Safari
  - `100vh` проблем на мобилни браузъри

### 2. Browser тестване — Функционалности
Тествай всяка функционалност в браузъра:

#### Кошница
- [ ] Добави продукт → badge се обновява
- [ ] Добави същия продукт 2 пъти → количеството расте
- [ ] Премахни продукт → цената се преизчислява
- [ ] Промени количество (+ / −) → правилна цена
- [ ] Количество 0 → продуктът се премахва
- [ ] Празна кошница → подходящо съобщение

#### Продуктов модал
- [ ] Отваряне → правилни данни (име, цена, снимка)
- [ ] Gallery навигация (← →) → снимките сменят
- [ ] Tab-ове (Описание, Спецификации, Ревюта) → правилно съдържание
- [ ] Quantity selector → работи правилно
- [ ] Добави от модал → добавя в кошница
- [ ] Затваряне (X, backdrop click, Escape) → затваря се

#### Търсене
- [ ] Въведи текст → live резултати
- [ ] Натисни Enter → филтрира продуктите
- [ ] Изчисти полето → връща всички продукти
- [ ] Търси несъществуващ продукт → "няма резултати"
- [ ] Специални символи (<script>) → няма XSS

#### Мобилно меню
- [ ] Hamburger → меню се отваря с анимация
- [ ] Категории accordion → разгъва се/свива
- [ ] Линк от менюто → навигира и затваря менюто
- [ ] Backdrop click → затваря менюто
- [ ] Body scroll lock → не може да scroll-ва зад менюто

#### Филтри
- [ ] Филтър по категория → показва правилните продукти
- [ ] Филтър по ценови диапазон → коректно
- [ ] Сортиране (цена ↑↓, име, рейтинг) → правилен ред
- [ ] Комбинация от филтри → работи

### 3. Edge Cases
- Тествай с празен products масив
- Тествай с много дълго име на продукт
- Тествай с цена 0 или отрицателна цена
- Тествай resize от desktop → mobile и обратно
- Тествай бърз double-click на бутони
- Тествай с disabled JavaScript (graceful degradation)

### 4. Console Monitoring
- Отвори браузъра и наблюдавай конзолата
- Логвай всички грешки и warnings при:
  - Зареждане на страницата
  - Навигация между секции
  - Взаимодействие с продукти
  - Отваряне/затваряне на модали

### 5. Създай Bug Report
- Създай artifact `bug_report.md` със:
  - 🔴 **Critical** — блокиращи бъгове
  - 🟠 **Major** — функционални проблеми
  - 🟡 **Minor** — козметични или edge case проблеми
  - 🟢 **Enhancement** — не е бъг, но може да е по-добре
  - За всеки бъг:
    - ID (BUG-001, BUG-002...)
    - Стъпки за възпроизвеждане
    - Очаквано поведение
    - Действително поведение
    - Скрийншот (ако е визуален)
    - Предложена корекция с конкретен код
    - Файл и ред, който трябва да се промени
