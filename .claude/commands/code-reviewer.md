# 🔍 Агент: Код Ревюър (Code Reviewer)

Този workflow анализира кодовата база за качество, поддръжка и потенциални проблеми.

## Стъпки

### 1. Статичен анализ на кода
- Прочети source файловете в `js/` (вж. `js/_load-order.txt` за пълния списък). `app.js` е генериран bundle — анализирай source файловете вместо него. Приоритизирай: `js/cart.js`, `js/auth.js`, `js/search.js`, `js/filters.js`, `js/ui.js`. Анализирай за:
  - Дублиран код (DRY violations)
  - Функции с прекалено много отговорности (>50 реда)
  - Неизползвани променливи и функции
  - Глобални променливи, които могат да се капсулират
  - Error handling — липсващи try/catch блокове
  - Memory leaks — event listeners, които не се премахват
  - Потенциални race conditions

- Прочети `styles.css` и анализирай:
  - Дублирани стилове
  - Неизползвани CSS класове
  - Липсващи media queries
  - CSS specificity проблеми
  - Inconsistent spacing/sizing

- Прочети `index.html` и анализирай:
  - Semantic HTML — правилна употреба на tags
  - Accessibility (a11y) — aria labels, alt текстове, focus management
  - Липсващи id атрибути за интерактивни елементи
  - Неправилно nested елементи

### 2. Архитектурен анализ
- Оцени общата архитектура на приложението
- Идентифицирай tight coupling между модулите
- Предложи разделяне на модули ако `app.js` е прекалено голям
- Анализирай data flow и state management подхода
- Провери за XSS уязвимости (innerHTML с потребителски вход)

### 3. Анализ на зависимости
- Прочети `package.json` за зависимости
- Провери за остарели или уязвими пакети
- Предложи допълнителни dev-tools (linter, formatter)

### 4. Създай Code Review Report
- Създай artifact `code_review.md` с:
  - 🔴 **Критични** — проблеми, които трябва да се оправят веднага (бъгове, сигурност)
  - 🟡 **Важни** — проблеми, които влияят на поддръжката
  - 🟢 **Препоръки** — nice-to-have подобрения
  - За всеки проблем: файл, ред, описание, предложена корекция
  - Обща оценка на кода (1-10)
  - Metric за техн. дълг

### 5. Предложи конкретни рефакторинги
- Дай конкретни code diff-ове за топ 5 критични проблема
- Обясни защо всяка промяна е важна
- Оцени risk/effort за всеки рефакторинг
