# 📋 Агент: Редактор на каталог (Data Editor)

Този workflow управлява продуктовия каталог в `products.js` — добавяне, редактиране и валидация.

## Стъпки

### 1. Разбери заявката
- Прочети `$ARGUMENTS` за да разбереш операцията:
  - Добавяне на нов продукт
  - Редактиране на цена/описание/снимка
  - Промяна на категория
  - Bulk update (напр. всички цени +10%)

### 2. Прочети текущата схема
- Прочети първите 5-10 продукта от `products.js` за да разбереш структурата
- Задължителни полета:
  ```javascript
  {
    id: Number,          // уникален, пореден
    name: String,        // пълно търговско наименование
    brand: String,       // марка
    category: String,    // трябва да съответства на съществуваща категория
    price: Number,       // в BGN, без ДДС
    oldPrice: Number,    // оригинална цена (за sale badge), може да е null
    image: String,       // URL или път
    images: Array,       // допълнителни снимки
    specs: Object,       // технически характеристики
    rating: Number,      // 0-5
    reviews: Number,     // брой ревюта
    inStock: Boolean,
    badge: String        // null | "Ново" | "Намаление" | "Топ продукт"
  }
  ```

### 3. Валидация преди промяна
- Провери за **дублирани ID** — `id` трябва да е уникален
- Провери дали `category` съществува в списъка с категории
- Провери дали `price` е положително число
- Ако добавяш продукт — използвай следващия свободен `id` (max(id) + 1)

### 4. Покажи преглед на промяната
Преди да запишеш, покажи:
```
Операция: Добавяне на продукт
Продукт: Samsung Galaxy S25 Ultra
ID: 247
Цена: 3499 BGN
Категория: Смартфони
```
Изчакай потвърждение ако промяната е голяма (bulk update на 10+ продукта).

### 5. Приложи промяната
- Редактирай `products.js` с точните промени
- Запазвай форматирането и indentation-а на файла

### 6. Тествай след промяната
// turbo
- Изпълни `npm test` — филтри и search тестовете използват products данни
- Провери, че новият продукт се показва в браузъра
- Провери, че search го намира по правилните ключови думи
- Провери, че филтрите го включват в правилната категория

## Формат на доклада
- Списък на всички направени промени (id, поле, стара стойност → нова стойност)
- `npm test` резултат
- Потвърждение за успешна валидация
