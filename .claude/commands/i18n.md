# 🌍 i18n Manager (Internationalization)

## Цел
Подготовка на сайта за многоезичност — извличане на hardcoded текстове, създаване на translation файлове и предложение за i18n архитектура.

## Стъпки

### 1. Одит на hardcoded текстове
- Сканирай source файловете в `js/` за Bulgarian hardcoded strings — фокусирай се на: `js/ui.js`, `js/cart.js`, `js/auth.js`, `js/actions.js`, `js/pages.js` (те съдържат повечето UI текстове). `app.js` е генериран bundle. Търси:
  - `showToast('...')` съобщения
  - Template literals с HTML (`innerHTML = '...'`)
  - Button labels, placeholder текстове
  - Error messages
  - Product category names
- Сканирай index.html за hardcoded текстове в:
  - Navigation links
  - Form labels и placeholders
  - Footer content
  - Modal headers и descriptions

### 2. Създай translation catalogue
Генерирай `locales/bg.json` с всички открити текстове:
```json
{
  "nav.home": "Начало",
  "nav.about": "За нас",
  "cart.empty": "Кошницата е празна.",
  "cart.add_success": "✓ {name} добавен!",
  "search.placeholder": "Търси по име, марка, модел...",
  "checkout.confirm": "Потвърди поръчката",
  "toast.error": "Нещо се обърка. Моля опресни страницата."
}
```

### 3. Създай English translation
Генерирай `locales/en.json` с преводи на всички текстове.

### 4. Предложи i18n архитектура
- `t()` helper функция за runtime translation lookup
- Language switcher компонент
- URL-based routing (`/en/`, `/bg/`)
- `<html lang="">` dynamic update
- `hreflang` meta tag update

### 5. SEO за многоезичност
- Отделен `<title>` и `<meta description>` per locale
- `hreflang` тагове за всеки език
- Sitemap с alternate links
- OpenGraph tags per locale

## Формат на доклада
- Списък на всички hardcoded текстове (grouped by component)
- Готов `bg.json` и `en.json`
- Архитектурно предложение за i18n implementation
