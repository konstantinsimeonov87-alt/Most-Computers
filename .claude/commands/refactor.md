# 🔧 Агент: Рефактор (Refactor)

Този workflow безопасно рефакторира source файловете в `js/` — с тестове, без счупване.

## Предварителни условия
- Трябва да има конкретна цел: кой модул/функция се рефакторира и защо
- Приеми целта като `$ARGUMENTS` (напр. "extract price formatting from js/cart.js")

## Стъпки

### 1. Разбери целта
- Прочети `$ARGUMENTS` за да разбереш какво трябва да се рефакторира
- Прочети целевия модул в `js/` изцяло
- Идентифицирай:
  - Точно кои функции/блокове ще се променят
  - Кои файлове ги импортват или използват
  - Дали има съществуващи тестове в `tests/`

### 2. Провери тестовото покритие
- Провери `tests/` за тестове на целевия модул
- Ако **няма тестове** за функциите, които ще се рефакторират:
  - Напиши минимален тест, който описва очакваното поведение
  - Провери, че тестът минава: `npm test -- --testPathPattern=<модул>`
  - Само тогава продължи с рефакторинга

### 3. Напиши рефактор план
Създай кратък план преди да пишеш код:
- Какво точно се променя (функция по функция)
- Какви рискове съществуват
- Как ще се тества след рефакторинга

### 4. Рефакторирай
- Редактирай само source файловете в `js/`
- **НЕ редактирай `app.js`** — той е генериран bundle
- Правило: едно нещо наведнъж (не рефакторирай 3 различни функции в един patch)
- Следвай съществуващите паттърни в кода:
  - `data-action` за event handling
  - CSS custom properties (`var(--primary)`)
  - SVG спрайт система (`#ic-*`)

### 5. Тествай
// turbo
- Изпълни `npm test` — всички тестове трябва да минават
- Ако има failing тестове: анализирай дали рефакторингът е счупил нещо или тестът е бил грешен
- Тествай в браузъра засегнатата функционалност

### 6. Rebuild bundle
- Изпълни `node build.js` за да обнови `app.js` с промените от `js/`
- Провери, че сайтът зарежда правилно след rebuild

### 7. Документирай
- Създай artifact `refactor_log.md` с:
  - Какво беше преди / след
  - Защо беше направена промяната
  - Тест резултати

## Формат на доклада
- Before/after code diff за ключовите промени
- `npm test` резултат (pass/fail count)
- Потвърждение, че `node build.js` е изпълнен успешно
