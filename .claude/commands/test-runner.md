# 🧪 Агент: QA Инженер (Test Runner)

Този workflow анализира кода, пише автоматични тестове и ги изпълнява.

## Стъпки

### 1. Анализирай какво трябва да се тества
- Прочети файловете в `js/` за да идентифицираш всички публични функции — вж. `js/_load-order.txt` за пълния списък от 18 модула. Всеки модул е отделен файл (`js/cart.js`, `js/search.js` и др.). `app.js` е генериран bundle — тестовете трябва да тестват source модулите.
- Прочети `products.js` за структурата на данните
- Прочети `index.html` за DOM структурата и data-action атрибутите
- Провери съществуващите тестове в `tests/` директорията
- Провери `jest.config.js` за текущата тест конфигурация

### 2. Категоризирай тестовете

#### 🔧 Unit тестове (Jest)
- Функции за форматиране на цени (BGN/EUR)
- Филтриране и сортиране на продукти
- Калкулации на кошница (цени, количества, отстъпки)
- Валидация на форми (бърза поръчка, регистрация)
- Search функционалност
- Wishlist операции
- Compare логика (добавяне, премахване, лимити)

#### 🌐 Browser тестове (Browser subagent)
- Зареждане на страницата — проверка за JS грешки в конзолата
- Кликване на продукт → отваряне на модал
- Добавяне в кошница → проверка на badge ъпдейт
- Мобилно меню → отваряне/затваряне
- Търсене → резултати се показват
- Responsive поведение на различни viewport-и

### 3. Напиши тестовете
- Създай нови тестови файлове в `tests/` директорията
- Именуване: `tests/[модул].test.js`
- Използвай `jest` и `jest-environment-jsdom` (вече инсталирани)
- За всеки тест: ясно описание какво тества и защо

### 4. Изпълни тестовете
// turbo
- Изпълни `npm test` в root директорията
- Ако има failing тестове, анализирай грешките
- Предложи fixes за счупени тестове или за кода, който тестовете разкриват като бъгав

### 5. Направи browser smoke test
- Използвай browser subagent за отваряне на сайта
- Провери за визуални бъгове
- Провери конзолата за грешки
- Направи скрийншотове на ключовите екрани

### 6. Създай Test Report artifact
- Създай `test_report.md` с:
  - Общ brой тестове и резултат (pass/fail)
  - Покритие по модули
  - Намерени бъгове с описание
  - Скрийншотове от browser тестовете
  - Препоръки за допълнителни тестове
