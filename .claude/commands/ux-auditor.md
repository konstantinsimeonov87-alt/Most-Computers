# 🎨 Агент: UX/UI Одитор (UX Auditor)

Този workflow анализира визуалния дизайн, потребителските потоци и цялостното UX на сайта.

## Стъпки

### 1. Визуален одит с browser
- Отвори сайта в браузъра
- Направи скрийншотове на всеки ключов екран:
  - Начална страница (hero, продуктови карти, footer)
  - Продуктов модал (gallery, specs, reviews)
  - Кошница (sidebar cart)
  - Мобилно меню
  - Форма за бърза поръчка
  - Модал за вход/регистрация
  - Страница за сравнение
- Тествай при различни viewport-и:
  - Desktop: 1920×1080, 1440×900, 1280×720
  - Tablet: 768×1024
  - Mobile: 375×812, 390×844, 320×568

### 2. UX Flow анализ
Тествай и документирай следните потребителски потоци:

#### 🛒 Покупков поток (Purchase Flow)
1. Разглеждане на продукти → Клик на продукт → Модал
2. Модал → Добави в кошница → Кошница се отваря
3. Кошница → Промяна на количество → Цената се обновява
4. Кошница → Checkout / Бърза поръчка

#### 🔍 Търсене (Search Flow)
1. Клик на Search → Въвеждане текст → Live резултати
2. Клик на резултат → Продуктов модал
3. Филтриране по категория

#### 📱 Mobile-First анализ
1. Hamburger меню → Навигация → Категории
2. Thumb-zone анализ — дали CTA бутоните са на достъпно място
3. Swipe gestures — дали работят carousel-ите
4. Touch target size — минимум 44×44px

### 3. Дизайн консистентност
- Провери цветовата палитра — дали е консистентна
- Провери типографията — font sizes, line heights, weights
- Провери spacing system — дали е consistent (4px/8px grid)
- Провери border-radius, shadows, transitions
- Провери dark mode (ако има) — дали всички елементи се виждат

### 4. Accessibility одит (a11y)
- Проверка за keyboard navigation (Tab, Enter, Escape)
- Контраст на цветовете (WCAG AA — 4.5:1 за текст)
- Screen reader friendly: aria-labels, роли, alt текстове
- Focus индикатори — дали са видими
- Skip-to-content линк

### 5. Конверсионен анализ
- Дали CTA бутоните са ясни и видими
- Дали има trust signals (гаранции, сертификати, отзиви)
- Дали цената е ясно представена
- Дали има urgency/scarcity елементи
- Friction points — къде потребителят може да се откаже

### 6. Създай UX Audit Report
- Създай artifact `ux_audit.md` със:
  - Скрийншотове от всеки екран (desktop и mobile)
  - Оценка по категории (1-10): Visual Design, Usability, Accessibility, Mobile UX, Conversion
  - Конкретни проблеми с визуални доказателства
  - Приоритизиран action plan за подобрения
  - Преди/След mockups за топ 3 проблема (използвай generate_image)
